WCAG 2.2 Compliance Matrix

This table maps Web Content Accessibility Guidelines (WCAG) 2.2 criteria to specific pages in this testbed that demonstrate violations.

Criterion Level Description Example Violation Page(s)
1.1.1 Non-text Content A All non-text content that is presented to the user has a text alternative that serves the equivalent purpose. images.html (Missing/Bad Alt Text)
1.3.1 Info and Relationships A Information, structure, and relationships conveyed through presentation can be programmatically determined or are available in text. structure.html (Headings, Lists)
tables.html (Layout Tables)
forms_basic.html (Missing Labels)
1.3.2 Meaningful Sequence A When the sequence in which content is presented affects its meaning, a correct reading sequence can be programmatically determined. focus_order.html (Flexbox vs DOM Order)
1.3.3 Sensory Characteristics A Instructions provided for understanding and operating content do not rely solely on sensory characteristics of components such as shape, color, size, visual location, orientation, or sound. forms_advanced.html (Color required asterisks)
1.4.1 Use of Color A Color is not used as the only visual means of conveying information, indicating an action, prompting a response, or distinguishing a visual element. contrast.html (Links color only)
forms_advanced.html (Color only errors)
1.4.3 Contrast (Minimum) AA The visual presentation of text and images of text has a contrast ratio of at least 4.5:1. contrast.html
1.4.10 Reflow AA Content can be presented without loss of information or functionality, and without requiring scrolling in two dimensions. zoom_responsive.html
2.1.1 Keyboard A All functionality of the content is operable through a keyboard interface without requiring specific timings for individual keystrokes. interactive.html (Mouse-only events)
keyboard_traps.html
2.1.2 No Keyboard Trap A If keyboard focus can be moved to a component of the page using a keyboard interface, then focus can be moved away from that component using only a keyboard interface. keyboard_traps.html
2.2.2 Pause, Stop, Hide A For moving, blinking, scrolling, or auto-updating information, there is a mechanism for the user to pause, stop, or hide it. flashing.html
2.3.1 Three Flashes or Below Threshold A Web pages do not contain anything that flashes more than three times in any one second period, or the flash is below the general flash and red flash thresholds. flashing.html
2.4.1 Bypass Blocks A A mechanism is available to bypass blocks of content that are repeated on multiple Web pages. index.html (Missing Skip Link)
iframes.html (Missing Title)
2.4.2 Page Titled A Web pages have titles that describe topic or purpose. index.html (Bad Title)
2.4.3 Focus Order A If a Web page can be navigated sequentially and the navigation sequences affect meaning or operation, focusable components receive focus in an order that preserves meaning and operability. focus_order.html
2.4.4 Link Purpose (In Context) A The purpose of each link can be determined from the link text alone or from the link text together with its programmatically determined link context. links.html ("Click Here")
2.4.6 Headings and Labels AA Headings and labels describe topic or purpose. structure.html (Fake Headings)
2.4.7 Focus Visible AA Any keyboard operable user interface has a mode of operation where the keyboard focus indicator is visible. focus_order.html (Outline: none)
2.5.3 Label in Name A For user interface components with labels that include text or images of text, the name contains the text that is presented visually. interactive.html (Implied labels on non-standard controls)
3.1.1 Language of Page A The default human language of each Web page can be programmatically determined. language.html
3.1.2 Language of Parts AA The human language of each passage or phrase in the content can be programmatically determined. language.html
3.3.1 Error Identification A If an input error is automatically detected, the item that is in error is identified and the error is described to the user in text. forms_advanced.html (Disconnected errors)
3.3.2 Labels or Instructions A Labels or instructions are provided when content requires user input. forms_basic.html
4.1.2 Name, Role, Value A For all user interface components (including but not limited to: form elements, links and components generated by scripts), the name and role can be programmatically determined. aria_bad.html
